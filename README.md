# lms-gamification-module

Модуль геймификации для российских LMS систем. 

Реализован как микросервисная Event-Driven архитектура (EDA) по следующим причинам: 

1. **Адаптивность к разнородным системам**. Российские LMS имеют разные API и архитектуры. Event-driven подход с адаптерами позволяет интегрироваться без изменения исходного кода LMS
2. **Масштабируемость**. Поскольку самые широко используемые ЛМС имеют 100,000+ пользователей, микросервисная архитектура позволяет масштабировать только нагруженные компоненты (например, обработку событий) и управлять нагрузкой
3. **Гибкость правил геймификации**. В образовательном контексте нужно управлять механиками и адаптировать правила под разные дисциплины, A/B тестировать эффективность геймификации
4. **Аналитика**. Можно собирать данные обо всех действиях студентов, анализировать эффекитвность разных правил, исследовать корреляцию между геймификацией и успеваемостью
